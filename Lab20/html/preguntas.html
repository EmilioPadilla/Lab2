¿Qué alternativas a jQuery existen?

Para empezar, cuándo debemos de usar una alternativa? Se usa cuando no necesitas todos los metodos que ofrece JQuery. Las alternativas son los siguientes.

  • Native JavaScript: Es el mejor porque no tienes que instalar nada. Tambien porque soporta navegadores viejos. Su sintaxis, sin embargo, no es tan amigable
  • Cash: Su sintaxis es similar a JQuery. Tiene todo lo basico de lo mismo.
  • UmbrellaJS: Es mas chico en cuanto a sus metodos. Aqui, puedes usar JS nativo en selectores Umbrella, a diferencia de JQuery.
  • ChibiJS: Creado para deshacerse del gran tamaño de jQuery, y tiene todos los metodos esenciales que este ofrece. No provee animaciones.

¿Qué es una promesa en js?

Se usa cuando el servidor promete que va a realizar una peticion. No implica que la vaya a realizar al momento, sino que ha recibido el request, y va a producir un valor unico. Ya sea un valor resuelto, o la razon por la cual no se resolvio. (i.e. un problema de conexion con la red). Dicha promesa puede tener 3 distintos estados:

  • Pending: Cuando el valor regresado no esta listo aun.
  • Fulfilled: Cuando y si el valor final esta listo. Este valor se asigna permanentemente con la promesa
  • Rejected: Si un error previno que el valor final fuera determinado.

¿Cómo funcionan async y await?

Async se usa antes de 'function', para indicar que la siguiente funcion va a funcionar de forma asincrona. Es decir, que no se va cargar junto con toda la pagina, sino que se van a mandar fragmentos de la aplicacion para que tanto el servidor como el cliente lo procesen de forma asincrona, y que no tengan que esperar a que todo la aplicacion cargue. La funcion siempre va a regresar una promesa al usar async. La palabra clave await determina que JS va a esperar hasta que la promesa regrese un resultado, para poder ejecutarse. Await significa JS esperará hasta que la promesa fije un valor.
