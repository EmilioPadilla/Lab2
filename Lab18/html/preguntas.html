¿Qué importancia tiene AJAX en el desarrollo de RIA's (Rich Internet Applications)?

Para empezar, AJAX es considerada como la opcion mas viable para crear RIA's. Este es un acronimo para Asynchronous Javascript and XML, la cual nos sirve para crear aplicaciones web que sean mas rapidas, mejoras, y mucho mas interactivas. Es decir, AJAX ha revolucionado la forma en la que el usuario interactua con el servidor para poder brindarle una experiencia mucho mas interactiva, y en consecuencia, con mayor fluidez, respuesta, y rapidez. AJAX, debido a su modelo, es capaz de mandar constantemente fracciones de requests al servidor para que este, de manera asincrona, pueda recibir la peticion, procesarla y mandarla, todo esto mientras seguimos interactuando con el sitio web, por lo que parece que nunca necesitamos esperar para seguir interactuando con el sitio en cuestion

¿Qué implicaciones de seguridad tiene AJAX? ¿Dónde se deben hacer las validaciones de seguridad, del lado del cliente o del lado del servidor?

Las implicaciones de seguridad que conlleva manejar AJAX no son mas complicadas que las que debemos de implementar dentro de aplicaciones que no son RIA's. Para empezar, ya que AJAX interactua con JS, CSS, HTML y PHP, es necesario seguir cumpliendo estas validaciones para que un malintencionado no logre su cometido. Por ejemplo, para JS, tenemos que seguir validando que no ocurra el XSS o el XSFR. De igual manera, por el lado de PHP y HTML, tenemos que asegurarnos de limpiar de manera constante los datos ingresados en html, con las herramientas que PHP nos ofrece, esto para evitar SQL Injection o ataques al cliente de manera directa. De igual manera, debemos de validar en todo momento que la sesion que busca utilizar el programa, sea el unico que pueda interactuar con el mismo, esto para evitar que un malintencionado llegue a la direccion del sitio de manera directa, saltandose paginas. Por ultimo, es importante mencionar las ultimas tecnologias, como los tokens, los cuales son un poco mas complicados, pero casi infalibles. Los tokens funcionan de manera que cuando el programador crea un formulario dentro de un sitio, este lleva consigo un token unico. Una vez que el cliente empieza a interactuar con el formulario y manda respuestas, el token unico acompaña al request hasta el servidor, el cual pasara si y solo si se encuentra con el mismo token que el generado.

¿Qué es JSON?

JSON corresponde a JavaScript Object Notation, el cual es un formato ligero para el intercambio de datos. Hoy en dia, es comunmente usado en aplicaciones RIA's que encesitan de informacion fraccionada para mandar al servidor. En esencia, solo es un subconjunto de la notacion literal de objetos de JS, pero debido a su alta implementacion para reemplazar XML, hoy en dia es considerado como un lenguaje independiente. Es importante mencionar que utiliza convenciones conocidas por programadores en la familia de lenguajes C, incluyendo C, C++, C#, Java, JavaScript, Perl, Python, y muchos otros. Este esta constituido por dos estrucutras: Las que corresponden a nombre-valor (conocido como objetos) y una lista ordenada de valores, que se implementan como arreglos, vectores, listas o secuencias.
