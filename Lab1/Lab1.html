URL

Un localizador de recursos uniforme, o por sus siglas en ingles 'uniform resource locator'. Sirve para sirve para apuntar a recursos que pueden variar en el tiempo.

MIME

Multipurpose internet Mail Expressions, es una serie de especificaciones dirigidas al intercambio de todo tipo de archivos atraves del internet, de forma transparente para el usuario. (Soluciones Via Internet, 2020)

Metodos de HTTP

Respuestas informativas
100 continue: Todo bien al cien.
101 Switching Protocol: Se acepta cambio de protocolo propuesto por usuario.
Respuestas satisfactorias
200 Ok: la solicitud ha tenido exito, varía por metodo.
202 Accepted: La solicitud se ha recibido, pero aun no ha actuado.
Redirecciones
302 Found: El recurso de la URI ha sido cambiado temporalmente.
308 Permanent Redirect: El recurso se encuenta permanentemente ahora en otra URI. (Mozilla, 2020)

Códigos de errores HTTP

Errores de cliente
401 Unauthorized y 403 Forbidden: En ambas el recurso no se puede mostrar, pero en la primera se puede ver la autenticacion y en la segunda no.
408 Request Timed Out: Enviada en una conexion inactiva en algunos servidores.
Errores de servidor
502 bad gateaway: Se obtuvo una respueta invalida al trabajar como puerta de enlace para manejar la peticion.
510 not extended: Extensiones adicionales son requeridas para que el servidor las cumpla. (Mozilla, 2020)

  • arrow_drop_down¿Cuál es la diferencia entre Internet y la World Wide Web?
    Esta es una común confusion, sin embargo, no debemos olvidarnos que el internet es la infraestructura que proporciona recursos necesario para que la World Wide Web funcione correctamente. Es decir, la www fue un invento creado por Berners-Lee con propositos militares, sin embargo, se percataron que este facilitaba la intercambio de información entre equipos físicos a largas distancias. La web, es especificamente el resultado de combinar le lenguaje HTML, el protocolo HTTP y la url. La internet, por su lado, se creó en 1969 en el momento en que hubo comunicación entre dos ordenadores conectados entre sí. La red informatica que posibilitó este intercambio se denominaba ARPANET, mas tarde incorporando protocolos TCP/IP, renombrándose como internet. (LINUBE, 2019)
  • arrow_drop_down¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    GET: Solicita representación de un recurso específico. Solo datos. Datos que viajan por el encabezado.
    HEAD: Solicita la representacion de un recurso específico, sin el cuerpo de la respuesta.
    POST: Enviar una identidad a un recurso específico, causando cambio en el estado del servidor. Se utiliza para enviar información sensible.
    PUT: Reemplaza todas las representaciones del recurso destino por la carga útil de la petición.
    PATCH: Aplicar modificaciones parciales a un recurso. Actualizar. (Puede estar bloqueado por el servidor)
    DELETE: Borra un recurso específico.(Puede estar bloqueado por el servidor) (Mozilla, 2020)
  • arrow_drop_down¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    Se debe de utilizar el método POST, pues este envía datos de manera "oculta", tal y como presenta la necesidad de usuarios y contraseñas. Caso contrario, el metodo GET envía los datos mostrandolos en el url generado.

  • arrow_drop_downUn servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    Significa que la solicitud ha tenido éxito, lo cual es almacenable de forma predeterminada. El 200 OK varía dependiendo del metodo de solicitud HTTP. (Mozilla, 2020)
  • arrow_drop_down¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    No necesariamente, la respuesta 404 Not Found indica que el servidor no ha podido encontrar el recurso solicitado. Estos son llamados vinculos rotos. Tecnicamente, el desarrollador debe solicitar que el contenido se vuelva a almacenar en los servidores o algun otro recurso de manera permanente.
    Culpa de usuario si tecleo mal el url.
    Culpa de programador si el recurso no esta en el servidor(Mozilla, 2020)
  • arrow_drop_down¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    La respuesta 500 representa un error interno del servidor, que significa que algo salió mal con el servidor pero no se puede ser más específico al respecto. Es decir, podria ser un error temporal del servidor que no necesita de correccion del desarrollador, pues no es un caso en especifico del url solicitado.
    Culpa del programador por no tener los recursos de manera correcta en el servidor. (Mozilla, 2020)
  • arrow_drop_down¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    Existen tags y atributos que han sido depreciados debido a que función, en la nueva versión de HTML5, se maneja mucho mejor con CSS. Algunos ejemplos de los tags son acronym, applet, big y frame. Algunos ejemplos de los atributos son reve, shape, profile, classid.
  • arrow_drop_down¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    HTML5 es una version más avanzada de HTML4, sin embargo, HTML sigue siendo mas compatible con algunos buscadores.
    Diferencias generales son una sintaxis mas simple y clara, elementos multimedia soportados nativamente, accesar a la ubicacion geografica del usuario, memoria caché extendida almacenandola en servidores SQL y multithreading con JS.
    De igual manera, HTML4 es considerado como codigo spaguetti (no sabes donde empieza ni donde termina)
    y HTML5 esta en formato lasagna (por capas).(Mozilla, 2020)

Todos los componentes de estilo se han depreciado y pasado a cambiarse mediante una hoja de estilo CSS. (Mozilla, 2020)

    ¿Cuál es el ciclo de vida de los sistemas de información?
    Es un enfoque por gases del analisis y diseño, para desarrollar de mejor manera mediante el uso de un ciclo especifico de actividades del analista y del usuario.
    El ciclo de vida clasico incluye investigacion preliminar, analisis, diseño, desarrollo, implementación, y mantenimiento. Se basa en la premisa de que hay un proceso continuo de mejoramiento. (Tecnologias Informacion, 2018)

    ¿Cuál es el ciclo de desarrollo de sistemas de información?
    Existen varios ciclos propuestos para el desarrollo de los sistemas o desarrollo de software, sin embargo, el mas comprendido y utilizado es el ciclo de vida en cascada.
    Se comprende de plan operativo, especificacion de requisitos, especificacion funcional, diseño, implementacion, integracion, validacion y verificacion, y mantenimiento.
    (Tutorials Point, 2020)
