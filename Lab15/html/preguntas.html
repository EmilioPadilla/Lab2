¿Por qué es una buena práctica separar el modelo del controlador?

Porque de esta manera te permite separar la manipulacion y extraccion de datos por parte de la base de datos en una entidad distinta a la que controla toda tu aplicacion. Dejando aparte de que siempre debemos de buscar que todas nuestras funciones sean atomicas, es decir, que cumplan con una y solo una funcion; el modelo vista controlador llegó para que podamos separar la idea de manejar e ingresar datos en la base de datos, y pedir y verificar los datos por parte del usuario.

¿Qué es SQL injection y cómo se puede prevenir?

Existen diferentes formas para asegurar que no caigamos en uno de los principales ataques que marca la OWASP -> SQL Injection. SQL Injection es la practica de inyectar codigo SQL que pueda alterar de cualquier manera las bases de datos de las que depende nuestra aplicacion, es decir, meter código correspondiente a las bases de datos que pueda condecernos privilegios, mostrarnos informacion, agregar relaciones que afecten el funcionamiento, o hasta eliminar una base de datos completa. Esto a traves de los formularios que usamos dentro de nuestra aplicacion para conseguir datos del usuario. Algunos metodos para prevenirnos son los siguientes:

  • Usar sentencias preparadas: Esta forma comprueba que el codigo que recibamos del usuario si corresponda con el tipo de dato que estamos buscanod.
  • Usar procedimientos almacenados: Limitar las entradas que pueda ingresar el usuario mediante un banco de respuestas pre diseñadas para el codigo
  • Validacion de input de WhiteList: Tener un set de caracteres válidos, y comparar constantemente con estos caracteres para saber si lo podemos usar
  • Sanitizar los inputs: Esto es, en sí, la combinacion de todas las practicas.
