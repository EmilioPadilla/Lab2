¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Como punto de partida, Java es un lenguaje de programación, mientras que no se puede decir lo mismo de JavaScript. Aun con esto, ambos comparten la misma sintaxis.
Lo anterior sucede debido a que JavaScript se integra en las páginas HTML y es interpretado (sin compilación) por el cliente. Lo que significa que la pagina web , que es el programa fuente, es el que se ejecuta. JavaScript esta basado en objetos, y Java es un lenguaje con paradigma orientado a objetos, existe diferencia. JavaScript se encarga de la declaracion de tipos de variables. De igual manera, las referencias a los objetos se comprueban en tiempo real durante la ejecucion del HTML. Mientras que en Java se hace al compilarse. (Rodriguez, 2005)

¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)

Los metodos del objeto Date se dividen entre getters y setters. Algunos ejemplos de getters son:

  • getTime(): regresa el numero de milisegundos desde 1 de Enero de 1970
  • getFullYear(): regresa el año en formato de 4 digitos
  • getDay(): regresa el numero del día de la semana (entre 0 y 6)

Algunos ejemplos de setters son:

  • setFullYear(): establece el año de un objeto Date
  • setHours(): establece la hora de un objeto Date (entre 0 y 23)
  • setTime(): establece el tiempo, en milisegundos, desde 1 de Enero de 1970
(W2Schools, 2020)
¿Qué métodos tienen los arreglos? (Menciona al menos 5*)

Los arreglos, a parte de ser dinámicos desde su creación, generan una serie de metodos que nos permiten modificarlos de a lo largo de su interpretación. Entre los metodos que existen, estan los siguientes:

  • toString(): regresa el numero de milisegundos desde 1 de Enero de 1970
  • join(): se comporta como toString() con la adición de un separador entre los elementos fruits.join("*")
  • pop(): remueve el ultimo elemento del array
  • shift(): remueve el primer elemento del array, desplazando cada localidad a n-1
  • push(): agrega un elemento en la ultima localidad del array
  • unshift(): como push, pero agrega a la primer localidad, y regresa el nuevo número de localidades dentro del arreglo.
¿Cómo se declara una variable con alcance local dentro de una función?

Se declara de forma tradicional, es decir, escribiendo la palabra clave var antes del nombre de la variable. La diferencia es que dicha variable se tiene que declarar en el cuerpo de la funcion, es decir, dentro de la misma.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Que la variable a utilizar conserve el valor asignado dentro de una funcion a lo largo de todo el documento. Dicho resultado lo podriamos usar a nuestro favor, o, por omisión, tenerlo en nuestra contra.
A nuestro favor, actualizando el estado de la varible de manera constante a lo largo de todo el documento, y usandola para relacionar distintos datos y operaciones necesarios para la ejecicion del programa.
En nuestra contra al alterar el valor de la variable (sin saberlo) y afectando resultados posteriores dentro del documento, o de igual manera, dentro de la función.
La variable let ha sido recientemente agregada a JavaScript, la cual nos sirve para permitir que una variable unicamente viva dentro de los corchetes de la funcion. Por su parte, var vive dentro de toda la funcion. Por ejemplo, cuando usamos un contador 'i' dentro de un ciclo for, si lo declaramos con var, podríamos volver acceder a ella ya que se encuentra dentro del codigo donde se ejecuta la funcion. Es decir, podríamos acceder a ella a traves de 'function.x', y cambiarla. Si la declaramos con let, esta muere tras terminar la funcion y no podemos acceder a ella con el ejemplo anterior.

Las regex son una secuencia de caracteres que podemos utilizar para trabajar efectivamente con Strings. De acuerdo a esta sintaxis, podemos buscar por un texto en un string, reemplazar un substring en un string, o extraer informacion de un string. Para buscar patrones con expresiones regulares dentro de un string, es necesaria la funcion
regex.test(str). Donde regex es una expresion que nosotros buscamos dentro del string, y es necesario definirla, por ejemplos
var regex = /hello/;. Se notara que el valor va dentro de diagonales, esto es NECESARIO. Siguiendo, str es el string dentro del cual estamos buscando la expresion regular.
