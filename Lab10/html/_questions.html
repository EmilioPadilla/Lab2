• arrow_drop_down¿Por qué es una buena práctica separar el controlador de la vista?
    Porque gracias a esta separacion, podemos lograr formar un patrón de diseño tipicamente conocido como MVC. En español este patrón se traduce como el paradigma de modelo-vista-controlador (Model-View-Controller paradigm). A lo que hace referencia es a separar la vista (la interaccion con el usuario), el modelo (el procesamiento de los datos), y el controlador (el ingreso de datos por parte del usuario). Si quisieramos hacer una analogia con cualquier sistema, entendemos que estos estan formados por tres capas: entrada-->procesamiento-->salida de datos; lo cual lo podemos entender como controlador-->modelo-->vista. Todo esto nos permite organizar la informacion que ingresa el usuario para que se pueda tomar una decision estructurada por medio del modelo, y asi darle la salida esperada de datos al usuario por medio del sistema versionador (Marston, 2004).
  • arrow_drop_downAparte de los arreglos $_POST y $_GET, ¿qué otros arreglos están predefinidos en php y cuál es su función?
    $_SERVER : Es un array que contiene informacion, tales como ubicacion, cabeceras y rutas de script.
    $_SESSION : Contiene variables de sesion, es decir, aquellas variables disponibles por el script actual.
    $_FILES : Contiene variables de subidas de ficheros, es decir, esta asociado a elementos subidos al script en curso mediante _POST.
    $_COOKIE : Variable tipo array asociado a las variables pasadas del script en curso a traves de las cookies.
    $_REQUEST : Por defecto contiene los contenidos de _GET, _POST, y _COOKIE.
    $_ENV : Variables pasadas por el script actual a traves del metodo del entorno.

  • arrow_drop_downQué es una cookie.
    Son pequeños pedazos de informacion guardados en el navegador en forma de texto. Son usadas para guardar informacion que ha sido guardada o cambiada en los diferentes sitios web para no tener que modificar todo lo que hagamos desde cero cada vez que entramos de nuevo a un sitio. Si fueran borradas, perderiamos todos los logs y configuraciones de las paginas web que usamos y tendriamos que reingresar y volver a configurar nuestro entorno.
  • arrow_drop_down¿Qué es XSS y cómo se puede prevenir?
    Cross Site Scripting, el cual es un tipo de vulnerabilidad en seguridad que se encuentra tipicamente en nuestras aplicaciones web. El XSS permite a los malintencionados inyectar informacion por medio del script lado del cliente, y mostrarla en las paginas web de los usuarios. Se puede evitar colocando la funcion 'html.specialchars'. De esta manera se omiten las llamadas de caracteres especiales dentro de las peticiones. y se elimina casi completamente el XSS.
