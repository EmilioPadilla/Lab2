¿Qué es ODBC y para qué es útil?

Corresponde a Conectividad de Bases de datos Abierta (Open Data Base Connectivity, por sus siglas en inglés), y es un estándar de acceso a las bases de datos desarrollado por SQL Access Group. Es útil debido a que sin él, no podríamos acceder a cualquier dato desde cualquier aplicacion, es decir, nos permite acceder a los archivos de las bases de datos sin importar qué sistema de Gestion de Bases de Datos esté almacenando los datos. La idea general de la funcion es proveer salidas y entradas homogeneas a las bases de datos, permitiendo que distintos controladores aseguren la conectividad de la aplicacion con las diferentes bases de datos. (Jac, 2005)



¿Qué es SQL Injection (SQi)?

Un atacante explota las vulnerabilidades obteniendo ventana de la forma en la que una base de datos realiza consultas. Malintencionados usan SQi para accesar a informacion sensible, modificar o crear nuevos permisos de usuario o manipular/destruir datos sensibles. Forma parte del Top 10 OWASP, y es por eso mismo que debemos de al menos prevenir con las recomendaciones generales para evitar este ataque.

¿Qué técnicas puedes utilizar para evitar ataques de SQL Injection?

Forma parte del Top 10 OWASP, y es por eso mismo que debemos de al menos prevenir con las recomendaciones generales para evitar este ataque. Entre estas, se encuentran:

  • Evitar construir consultas dinamicas, es mejor utilizar declaraciones preparadas con consultas parametrizadas.
  • Usar procedimientos guardados, usando sintaxis canónica.
  • Sanitizar datos del usuario, usando htmlspecialchars()
  • Suprimir mensajes de error por parte de la base de datos, para evitar revelar errores que un atacante puede explotar
  • Limitar los permisos al usuario de la aplicacion, siguiendo el principio de menos privilegios
  • Usar soluciones SAST (Static Application Security Testing)
