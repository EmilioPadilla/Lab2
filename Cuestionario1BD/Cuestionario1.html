Cuestionario 1: BD y BDMS

  • arrow_drop_down¿Qué es entonces una base de datos - DB?
    Coleccion de archivos interrelacionados que, para no caer en archivos supeditados a programas o individuos, cumplen con restricciones que llevan a un mínimo de repeticion de datos. Por supuesto, siempre cumpliendo las necesidades de la empresa. Ej: Una base de datos que represente a los alumnos puede tener registros de matricula, nombre, direccion y el telefono, de manera qeu esto represente al alumno. Si bien, lo anterior suena como cualquier archivo, su practicidad reside en el administrador de bases de datos. (Juarez, 2020)
  • arrow_drop_down¿En qué casos es conveniente usar bases de datos?
    Aquellas situaciones en las que haya varios archivos interrelacionados, siendo estos requeridos para usuarios apliaciones. Un factor decidivo es el volumen de los datos, donde, si el numero de registros es grande, es muy conveniente almancenarlo en una base de datos. (Juarez, 2020)
  • arrow_drop_down¿Qué es un sistema de gestión de base de datos - DBMS?
    Data Base Management System, es aquel sistema que consiste en una coleccion de datos relacionados y un conjunto de programas para acceder a esos datos. Tiene que ser conveniente y eficiente para extraer y almacenar informacion en la base de datos. Esto incluye definicion de estructuras para almacenamiento de informacion, es decir, mecanismos de gestion de informacion. De igual manera, deben garantizar la seguridad de los datos, pese a caidas o intentos de acceso no autorizados. (Silberschatz, Korth, Sudarshan, 2006)
  • arrow_drop_downEnlista y explica las funciones/responsabilidades que tienen los DBMS.
    Su objetivo primordial es simplicar y facilitar el acceso a los datos. Es decir, el funcionamiento del sistema depende la eficiencia de las estructuras de datos usadas para representar los datos en la base de datos y de la capacidad de eficiencia para operar sobre dichas estructuras de datos.
    Entonces, el gestor de base de datos es responsable Interaccion con el gestor de archivos y traduccion de distintas sentencias DML a comandos del sistema de archivos de bajo nivel. Dicho de otra manera, lo podemos enlistar en lo siguiente:
    Implantacion de integridad => Se refiere a restricciones de consistencia de datos. Y acciones consecuentes de la restriccion.
    Implantacion de seguridad => Se refiere a RBAC a nivel datos, siendo trabajo del gestor cumplir con estos requisitos.
    Copia de seguridad y recuperacion => Debido a la probabilidad de fallos (perdida de informacion de datos por roturas de disco, fallas de software, etc.) es responsabilidad del gestor restaurar la base de datos al estado que tenía antes de ocurrir el fallo. Ocurre gracias a la iniciacion de procedimientos de copias de seguridad y recuperacion.
    Control de concurrencia => Controlar interaccion entre usuarios concurrentes, de lo contrario, se perderia consistencia. (Silberschatz, Korth, Sudarshan, 2006)
  • arrow_drop_downEjemplifica y justifica en que proyecto de los que has realizado hubiera sido conveniente utilizar una base de datos.
    Proyecto final en C para fundamentos de programación. - Se usaron archivos para crear un registro de las recetas disponibles dentro del catalogo. El sistema era capaz de buscar recetas existentes, de agregar nuevas recetas, y de catalogar. Las bases de datos hubieran servido para simplificar la insercion, modifiacion y eliminacion de recetas. Las bases de datos podrían haber ampliado el catálogo, ademas de simplificar la busqueda de recetas, asi como la modificación de las mismas.
    Proyecto final de desarrollo de videojuegos - Se usaron archivos para guardar los mapas que se crearan a lo largo del videojuego. De igual manera, guardabamos los datos de los jugadores que llegaban a interactuar con el videojuego. Las bases de datos hubieran sido de ayuda para simplificar el acceso a los jugadores regitrados, asi como la actualizacion de los mismos datos mediante estructuras que optimicen sus modificaciones.
